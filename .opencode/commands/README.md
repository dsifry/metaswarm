# OpenCode commands — generated output

**Do not hand-edit files in this directory.** Every `<name>.md` here is
regenerated by `lib/sync-resources.js` from `TOML_COMMAND_MAP`. Any local
edits will be silently overwritten the next time anyone runs:

```bash
node lib/sync-resources.js --sync
```

(which is enforced by `lib/sync-resources.js --check`, run in CI).

## To change a command

1. Edit the entry in `TOML_COMMAND_MAP` in `lib/sync-resources.js`.
2. Run `node lib/sync-resources.js --sync`.
3. Commit both the source change and the regenerated `.md` file(s).

The same source map also drives `commands/metaswarm/<name>.toml` (Gemini)
so a single edit propagates to every CLI host.

## To add a new command

Add a new entry to `TOML_COMMAND_MAP` with `description` and `prompt` fields,
then `--sync`. The generator handles `{{args}}` → `$ARGUMENTS` translation
for OpenCode automatically.

## Why not author directly here?

OpenCode, Gemini CLI, and (potentially) future CLIs have different
slash-command file formats. Keeping one source map and N format generators
ensures the command catalog stays identical across every CLI metaswarm
supports.
